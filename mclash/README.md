# Mclash App Source

This directory contains the Flutter application and Android host project for
Mclash. User-facing build and release instructions live in the repository root.

## Layout

```text
lib/
android/app/src/main/
android/app/src/debug/
android/app/src/profile/
android/app/src/main/kotlin/com/liuyihtu/mclash/
android/app/src/main/kotlin/hev/htproxy/
android/app/src/main/jniLibs/
android/key.properties
```

- `lib/` contains the Flutter entry point and UI pages.
- `android/app/src/main/` contains Android manifests, resources, VPN service,
  Quick Settings tile support and native bridge code.
- `android/app/src/main/kotlin/com/liuyihtu/mclash/` contains the app-specific
  Kotlin sources.
- `android/app/src/main/kotlin/hev/htproxy/` contains the Android service glue
  for HevSocks5Tunnel.
- `android/app/src/main/jniLibs/` is generated by the build workflow and holds
  `libmihomo.so` plus `libhev-socks5-tunnel.so`; it is intentionally ignored by
  Git.
- `android/key.properties` is an optional local signing file and must not be
  committed.

## Local Checks

From this directory:

```bash
flutter pub get
flutter analyze
```

Android release builds also require the native libraries generated by the root
GitHub Actions workflow or an equivalent local native build. Without
`android/key.properties`, release APKs are built unsigned. With a complete local
`android/key.properties`, Gradle applies the release signing config.

The Android package and namespace are both `com.liuyihtu.mclash`.
